## 0. Reviewer verdict

**Agreed:** the reuse-first thesis is correct and is now the law of this repo —
*enrich existing live JSON in place, add envelope/lifecycle/ledger metadata, fix
the lifecycle + staging bugs, and never fork a `*_latest.json` beside a file the
app already reads.* The audit (`reports/destination_contract_baseline.md`) proves
it: most destinations are already migrated.

**Two corrections to the reviewed draft (verified in code):**

1. **Insight is already `schemaVersion: 3`, not 2.** `insight_json_contract.py`
   sets `schemaVersion = 3`; the adapter gate is
   `SUPPORTED_SNAPSHOT_SCHEMAS = new Set([2, 3])`

   unsupported. The JSDoc "schemaVersion-2" comment is stale. **Invariant:**

   Set + cert tests.

2. **Buzz is not greenfield — it is DERIVED from Sections.**
   `buzzDataset.load()` builds trends from `loadSectionsDataset()` plus a live
   `rssAggregator` fallback. There is **no seed file and no greenfield gap.**
   So `buzz_latest.json` is **optional enrichment with a mandatory fallback to
   the derived path**, exactly like Planner and Following — not "the one true
   new pipeline." Net: **0% greenfield.**

The audit registry + tests in this PR were corrected accordingly (Buzz →
`DERIVED`; `NOT_MIGRATED` count is now 0).

## 2. Standing guardrails (hard rules, enforced by tests)

- **G1 — Schema bumps are gated by the adapter set.**
  Sections stays **schema 2** (adapter `{1,2}` *throws*). Insight stays **schema
  3** (adapter `{2,3}`). Never emit a version outside the adapter's Set without
  updating the adapter + its `.cert.test.js` in the same PR.
- **G2 — No forked siblings.** Never create `weather_latest.json`,
  `market_latest.json`, `newspaper_latest.json`, or `data_health_dashboard.json`.
  Enrich `weather_snapshot.json` / `market_snapshot.json` / `epaper_data.json` /
  `quality_dashboard.json` in place. (`buzz_latest.json` / `planner_latest.json`
  / `following_latest.json` are allowed: no existing file to fork — but each is
  **fallback-optional**.)
- **G3 — Every committed output is staged, proven by the audit.** New/regenerated
  committed JSON must be added to its workflow `git add`. PR exit gate: audit
  "committed output not staged" warnings must be **0**.
- **G4 — Envelope/clusters/lifecycle are additive.** All existing
  `*.cert.test.js`, `scripts/test_*` suites stay green. Consumers must work with
  *and without* the new fields.
- **G5 — Lifecycle families are separate (never `news_36h` for non-news).**
  Forecast→`validUntil`; market→market `validUntil`; alert→`expiryAt|+24h`;
  offer/shopping→`saleEndAt|+48h`; event→`[-1d,+7d]`.
- **G6 — Optional indices fall back.** Deleting `planner_latest.json` /
  `buzz_latest.json` / `following_latest.json` must not break the page (it
  re-derives). Proven by a "delete-the-file" test.
- **G7 — Repo conventions only.** Python → `scripts/test_*.py`; JS/UI →
  `scripts/test_*_static.mjs` + `src/**/*.cert.test.js`. **No new root `tests/`
  Python tree.** Certification extends `scripts/run_certification_gate.mjs` +
  `scripts/certification_manifest.json`.
- **G8 — Never touch device-local state from a collector.** `user_plan.json`
  (saved planner) is owned by the browser. The planner builder reads
  `up_ahead.json` only.
