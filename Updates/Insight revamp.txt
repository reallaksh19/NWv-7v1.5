# Plan: Insight C+E Architecture — Event Evolution + Source Taxonomy + Information Delta
## Elaborated for implementation by any engineer

---

## GOVERNING PRINCIPLE

> Selection should optimize **incremental information gain**, not just label diversity.
> Without delta scoring, C+E becomes source-bucket + time-bucket diversity — not Insight.

**The core model shift:**
- Old model: "pick stories from different sources and angle labels"
- New model: "pick stories that collectively tell the complete evolution of an event — what happened, what changed, who responded, what it means"

**Why the old model fails:** 62% of stories get `base_report` because the keyword-based
`classifyAngle()` only fires on signal words ("said", "markets react", etc.). Wire copy
has no such signals. So 5 Reuters stories about the same crash all get `base_report`
and the selector has nothing to diversify.

**Why the new model works:** Every story has a `publishedAt` timestamp and a `sourceGroup`.
Time + domain type give you reliable diversity signals without keyword dependency.
Information delta adds the quality signal: prefer the story that adds new facts.

## CRITICAL EXECUTION CORRECTIONS (read before implementing anything)

### 1 — Pipeline ordering (Agent 1 — highest priority)

The plan assigns `temporalTier`, `evolutionRole`, and `informationDelta` AFTER
`buildChildTree()`. That is wrong. `buildChildTree()` calls `computeInformationGain()`
internally — if C+E fields aren't populated yet, the first selection pass runs on
stale data.

**Correct order inside `selectTopParentsWithWeakTreeCheck`:**
```
clusterStories
  → assign temporalTier          ← MUST COME FIRST
  → assign evolutionRole
  → extractInformationDelta
  → applyPostClusterDeltaDedup
  → buildChildTree()             ← can now use delta/role in gain computation
  → enforceSourceDiverseChildSelection (C+E two-pass)
  → final display ordering
```

### 2 — Shadow mode before flipping behavior (Agent 0)

Never flip `buildChildTree` → C+E selector in production without a comparison run.
Feature flags control the rollout:

```
INSIGHT_CE_MODEL=off|shadow|on      (default: off during development, shadow before PR merge)
INSIGHT_SELECTOR_MODE=legacy|ce     (default: legacy; only flip after shadow validates)
INSIGHT_QUALITY_STRICT=0|1
```

In `shadow` mode: run BOTH selectors, write a comparison report, keep legacy output
for UI. In `on` mode: use C+E output. This lets you validate without risking the UI.

### 3 — Bridge computeInformationGain, don't replace it (Agent 2)

`buildChildTree` calls `computeInformationGain()` at `treeBuilder.ts:619`.
Do NOT delete this function. Bridge it:

```typescript

  selected: InsightStory[],
```

This makes the new model take effect inside `buildChildTree` automatically once

### 4 — C+E enrichment must use actual ordering in the cluster (Agent 5 + Agent 6)

`extractInformationDelta(story, baseline)` must receive stories sorted by
`publishedAt` ascending where `baseline = stories with publishedAt < story.publishedAt`.
If you pass the whole cluster unsorted as baseline, every story sees itself as "new."

## PHASE C + 1B — Source Taxonomy + Metadata Normalisation

### Context and motivation

The live data has 703 stories. 433 (62%) are labelled `base_report`. Why?
Because `classifyAngle()` is keyword-based: it only fires on signal words.
Most news wire copy is plain declarative text with none of these signals.

But the data already contains structural signals we're ignoring:
- `sourceGroup: "economic_times"` — this IS a business angle signal, no keywords needed
- `sourceGroup: "gadgets360"` — this IS a tech angle signal
- `category: "Markets"` on a Reuters article — this IS a business angle signal

Phase C wires these up.

**The critical split:** A source can be BOTH wire (how it distributes) AND business
(what it covers). Reuters carries market reports — they're wire-distributed but
business-domain. Do NOT treat Reuters market pieces as general-news low-diversity.

```typescript
export function getSourceDistributionType(sourceGroup: string): SourceDistributionType {
  return SOURCE_DISTRIBUTION_MAP[sourceGroup.toLowerCase().trim()] ?? "unknown";
}

export function getSourceContentDomain(
  sourceGroup: string,
  category?: string
): SourceContentDomain {
  // Priority 1: article-level section/category
  if (category) {
    const catKey = category.toLowerCase().trim().replace(/[^a-z]/g, "");
    const fromCategory = SECTION_DOMAIN_MAP[catKey] ?? SECTION_DOMAIN_MAP[category.toLowerCase().trim()];
    if (fromCategory) return fromCategory;
  }
  // Priority 2: source-level domain
  const fromSource = SOURCE_CONTENT_DOMAIN_MAP[sourceGroup.toLowerCase().trim()];
  if (fromSource) return fromSource;
  // Priority 3: regional heuristic — if source name contains a state/city
  const regional = /chennai|mumbai|delhi|bengaluru|kolkata|hyderabad|pune|tamil|kerala|andhra|telangana|rajasthan|gujarat|odisha|oman|muscat/i;
  if (regional.test(sourceGroup)) return "regional";
  return "general";
}

export function computeTrustScore(tier: SourceTier, distributionType: SourceDistributionType): number {
  const base = TIER_AUTHORITY[tier];
  const wireBonus = distributionType === "wire" ? -0.1 : 0;  // wire is less authoritative for diversity
  return Math.max(0, Math.min(1, base + wireBonus));
}
```

```typescript
function isDiversityProtectedTierC(
  story: InsightStory,
  higherTierStories: InsightStory[]
): boolean {
  // A business or regional story survives even at Tier C
  // (they're often mis-tiered or are legitimately niche)
  if (story.sourceContentDomain === "business") return true;
  if (story.sourceContentDomain === "regional") return true;
  if (story.sourceContentDomain === "official") return true;
  if (story.sourceContentDomain === "legal") return true;

  // A story survives if it adds a new content domain to the cluster
  const existingDomains = new Set(higherTierStories.map(s => s.sourceContentDomain));
  if (!existingDomains.has(story.sourceContentDomain)) return true;

  return false;
}
```

**Why this matters:** Without this fix, every business/regional story at Tier C gets
silently dropped from any parent that has a Reuters/Hindu story. The domain taxonomy
work in C-1 through C-3 would be completely nullified. The fix ensures that diversity-
adding Tier C stories survive the fallback filter.

```typescript
import { InsightParent, InsightStory, TemporalTier, EvolutionRole } from "../types";

// ── Re-anchor logic ───────────────────────────────────────────────────────────

/**
 * Computes the anchor time for temporal tier classification.
 *
 * For ACUTE events (train crash, election result, breaking news):
 *   anchor = parent.firstSeenAt   (the actual event start)
 *
 * For CHRONIC/ROLLING events (ongoing war, political scandal, slow-moving case):
 *   anchor = Date.now() - windowHours * 3600_000  (rolling 48h window)
 *
 * Why re-anchor? If a story about an ongoing war has parent.firstSeenAt = 3 months ago,
 * ALL stories would get temporalTier = "aftermath" regardless of their content.
 * Re-anchoring to the capture window makes the tier reflect "position in the current
 * coverage cycle", not "time since the war started".
 */
export function computeEventAnchor(
  parent: InsightParent,
  windowHours: number
): number {
  const windowStart = Date.now() - windowHours * 3600_000;
  // If the event started within the capture window, use actual start
  if (parent.firstSeenAt >= windowStart) return parent.firstSeenAt;
  // Otherwise re-anchor to the start of the capture window
  return windowStart;
}

// ── Temporal tier from delta hours ───────────────────────────────────────────

export function computeTemporalTier(
  publishedAt: number,
  anchor: number
): TemporalTier {
  const deltaHours = (publishedAt - anchor) / (1000 * 60 * 60);
  if (deltaHours < 0) return "breaking";     // published before anchor = breaking context
  if (deltaHours < 1) return "breaking";
  if (deltaHours < 4) return "developing";
  if (deltaHours < 12) return "established";
  if (deltaHours < 24) return "reaction";
  if (deltaHours < 36) return "analysis";
  return "aftermath";
}

// ── Evolution role inference (heuristic — Phase A replaces with LLM) ─────────

/**
 * Assigns an editorial evolution role to a story.
 *
 * KEY INSIGHT: temporalTier gives a rough time bucket, but the actual role
 * can differ. An official confirmation may come at 30h (analysis tier) for
 * an election result. A correction may come at any time.
 *
 * Strategy: Use signal words and source domain as overrides on the temporal prior.
 */
export function inferEvolutionRole(
  story: InsightStory,
  tier: TemporalTier,
  clusterBaseline: InsightStory[]
): EvolutionRole {
  const text = `${story.title} ${story.summary}`.toLowerCase();

  // High-confidence overrides (signal-word based):
  if (story.correctionMarker) return "fact_update";

  if (/\b(arrested?|detained?|charged?|convicted?|sentenced?)\b/.test(text))
    return "legal_or_regulatory";

  if (/\b(inquiry|investigation|probe|cbi|ed |enforcement)\b/.test(text))
    return "investigation";

  if (/\b(said|says|announced|confirmed|statement|approved|rejected|ministry|government|official)\b/.test(text) &&
      tier !== "breaking")
    return "official_response";

  if (story.sourceContentDomain === "business")
    return "market_reaction";

  if (story.sourceContentDomain === "regional")
    return "background_context";

  // Temporal fallback — when no signal words, use time bucket as role proxy
  switch (tier) {
    case "breaking":    return clusterBaseline.length === 0 ? "first_report" : "corroboration";
    case "developing":  return "fact_update";
    case "established": return "official_response";
    case "reaction":    return "public_reaction";
    case "analysis":    return "background_context";
    case "aftermath":   return "accountability";
  }
}
```

### E-3: Two-stage dedup reform in `dedup.ts`

**Context:** The current `removeHardDuplicates` runs globally O(n²) before clustering.

```typescript
/**
 * Post-cluster dedup: hides stories that add no new information compared to
 * stories already selected in the parent cluster.
 *
 * Called AFTER temporalTier and evolutionRole are assigned.
 * Called BEFORE child selection.
 *
 * Rules:
 * 1. Wire stories that are near-identical to earlier cluster stories get hidden
 *    (wire services re-stamp identical copy with updated timestamps)
 * 2. Stories with no delta AND same evolution role as an existing story get hidden
 * 3. Stories with important roles (official_response, market_reaction, cause_claim,
 *    investigation) are NEVER hidden by post-cluster dedup
 */
export function applyPostClusterDeltaDedup(
  clusterStories: InsightStory[],
  hiddenIds: Set<string>,
  cfg: InsightConfig
): InsightStory[] {
  const PROTECTED_ROLES = new Set<EvolutionRole>([
    "official_response", "market_reaction", "cause_claim",
    "cause_confirmed", "investigation", "legal_or_regulatory", "accountability",
  ]);

  // Sort by publishedAt ascending — earlier stories establish the baseline
  const sorted = [...clusterStories].sort((a, b) => a.publishedAt - b.publishedAt);
  const kept: InsightStory[] = [];

  for (const story of sorted) {
    // Always keep the first story
    if (kept.length === 0) { kept.push(story); continue; }

    // Never hide protected evolution roles
    if (story.evolutionRole && PROTECTED_ROLES.has(story.evolutionRole)) {
      kept.push(story);
      continue;
    }

    // Wire cross-tier dedup: if wire-distributed AND near-identical to earlier story
    if (story.sourceDistributionType === "wire" && cfg.WIRE_DUP_CROSS_TIER) {
      const wireDuplicate = kept.find(k =>
        k.sourceGroup === story.sourceGroup &&
        titleSimilarity(k.title, story.title) >= 0.90
      );
      if (wireDuplicate) {
        hiddenIds.add(story.id);
        continue;
      }
    }

    // Low-delta no-new-role: hide if same role AND low delta score
    if (story.evolutionRole === "no_new_information" ||
        (story.informationDeltaScore !== undefined &&
         story.informationDeltaScore < 0.10)) {
      const sameRole = kept.find(k => k.evolutionRole === story.evolutionRole);
      if (sameRole) {
        hiddenIds.add(story.id);
        continue;
      }
    }

    kept.push(story);
  }

  return kept;
}
```

### Phase 3B: New file — `src/insight/src/pipeline/deltaAtoms.ts`

Create from scratch:

```typescript
import { InsightStory, InformationDelta } from "../types";

/**
 * Computes what is NEW in `story` compared to the `baseline` stories.
 *
 * deltaScore: weighted count of new signals.
 *   - New numbers: 0.35 weight (highest — these are facts)
 *   - New named entities: 0.25
 *   - New event verbs: 0.15
 *   - New places: 0.15
 *   - Correction signal: 0.10
 *
 * A deltaScore of 0 means the story adds nothing new.
 * A deltaScore of 1 means the story adds a large amount of novel information.
 */
export function extractInformationDelta(
  story: InsightStory,
  baseline: InsightStory[]
): InformationDelta {
  if (baseline.length === 0) {
    // First story — everything is new
    return {
      newNumbers: story.numbers,
      newEntities: [
        ...story.entities.people,
        ...story.entities.orgs,
      ],
      newPlaces: story.entities.places,
      newVerbs: story.eventVerbs,
      newAttributions: [],
      newClaims: [],
      changedNumbers: [],
      correctionSignals: story.correctionMarker ? ["correction"] : [],
      deltaScore: 1.0,
    };
  }

  // Collect what's already known from baseline stories
  const knownNumbers  = new Set(baseline.flatMap(s => s.numbers).map(n => n.toLowerCase()));
  const knownEntities = new Set(baseline.flatMap(s =>
    [...s.entities.people, ...s.entities.orgs]).map(e => e.toLowerCase())
  );
  const knownPlaces   = new Set(baseline.flatMap(s => s.entities.places).map(p => p.toLowerCase()));
  const knownVerbs    = new Set(baseline.flatMap(s => s.eventVerbs).map(v => v.toLowerCase()));

  const newNumbers  = story.numbers.filter(n => !knownNumbers.has(n.toLowerCase()));
  const newEntities = [...story.entities.people, ...story.entities.orgs]
    .filter(e => !knownEntities.has(e.toLowerCase()));
  const newPlaces   = story.entities.places.filter(p => !knownPlaces.has(p.toLowerCase()));
  const newVerbs    = story.eventVerbs.filter(v => !knownVerbs.has(v.toLowerCase()));

  const correctionSignals = story.correctionMarker ? ["correction"] : [];

  // Changed numbers: a number in this story that overrides a known number
  // (heuristic: same numeric key with different value — simplified for Phase E)
  const changedNumbers: Array<{key: string; from?: string; to: string}> = [];

  // Raw scores (count-based, normalized)
  const numberScore     = Math.min(1, newNumbers.length * 0.35);
  const entityScore     = Math.min(1, newEntities.length * 0.15);
  const placeScore      = Math.min(1, newPlaces.length   * 0.15);
  const verbScore       = Math.min(0.3, newVerbs.length  * 0.10);
  const correctionBonus = correctionSignals.length > 0 ? 0.20 : 0;

  const deltaScore = Math.min(1.0,
    0.35 * numberScore +
    0.25 * entityScore +
    0.15 * placeScore  +
    0.15 * verbScore   +
    0.10 * correctionBonus
  );

  return {
    newNumbers,
    newEntities,
    newPlaces,
    newVerbs,
    newAttributions: [],
    newClaims: [],
    changedNumbers,
    correctionSignals,
    deltaScore,
  };
}
```

## PHASE 4 — C+E Two-Pass Child Selector

### Context

Now that every story has `temporalTier`, `evolutionRole`, `informationDeltaScore`,
`sourceContentDomain`, and `sourceDistributionType`, we can replace the current
source-diversity selector with one that optimizes information gain.

**Old selector:** "pick stories that add a new source group, score by quality"
**New selector:** "pick stories that tell the next chapter of the event's evolution"

### Rewrite `enforceSourceDiverseChildSelection` in `sourceDiverseChildSelection.ts`

The function signature is unchanged. Only the internals change.

```typescript
export function enforceSourceDiverseChildSelection(
  parent: InsightParent,
  selectedChildren: InsightStory[],
  clusterStories: InsightStory[],
  cfg: InsightConfig,
  hiddenIds: Set<string>
): InsightStory[] {

  // ── Candidate scoring ─────────────────────────────────────────────
  function computeCellKey(story: InsightStory): string {
    return `${story.sourceGroup}:${story.sourceContentDomain}:${story.temporalTier}:${story.evolutionRole}`;
  }

  function computeDomainDiversityBonus(story: InsightStory): number {
    const domainWeight = cfg.DOMAIN_DIVERSITY_WEIGHT[story.sourceContentDomain] ?? 0.25;
    const wirePenalty  = story.sourceDistributionType === "wire"
      ? cfg.WIRE_DOMAIN_DIVERSITY_PENALTY
      : 0;
    return Math.max(0, domainWeight - wirePenalty);
  }

  function computeCandidateScore(
    story: InsightStory,
    selected: InsightStory[]
  ): number {
    const coveredTiers  = new Set(selected.map(s => s.temporalTier));
    const coveredRoles  = new Set(selected.map(s => s.evolutionRole));
    const coveredGroups = new Set(selected.map(s => s.sourceGroup));
    const coveredDomains = new Set(selected.map(s => s.sourceContentDomain));

    const isNewTier   = story.temporalTier   ? !coveredTiers.has(story.temporalTier)   : false;
    const isNewRole   = story.evolutionRole  ? !coveredRoles.has(story.evolutionRole)  : false;
    const isNewGroup  = !coveredGroups.has(story.sourceGroup);
    const isNewDomain = !coveredDomains.has(story.sourceContentDomain);

    const deltaScore  = story.informationDeltaScore ?? 0.5;
    const penalty     = story.repeatedFactPenalty   ?? 0;
    const authority   = Math.min(1, (story.sourceAuthority ?? 0) + (story.trustScore ?? 0) * 0.2);

    return (
      0.30 * deltaScore                          // PRIMARY: does this add new facts?
    + 0.25 * (isNewRole   ? 1 : 0)             // editorial distinctness
    + 0.18 * (isNewTier   ? 1 : 0)             // temporal diversity
    + 0.12 * computeDomainDiversityBonus(story) // business/regional bonus
    + 0.08 * authority                          // source credibility
    + 0.07 * (isNewGroup  ? 1 : 0)             // confirmation breadth
    - 0.25 * penalty                            // penalise repeated facts
    );
  }

  // ── Reject by cell key ─────────────────────────────────────────────
  function shouldReject(story: InsightStory, selected: InsightStory[]): boolean {
    const key = computeCellKey(story);
    const sameCell = selected.find(s => computeCellKey(s) === key);
    // Same cell AND low delta = reject (this is a near-duplicate in disguise)
    return sameCell !== undefined && (story.informationDeltaScore ?? 1) < 0.10;
  }

  // ── Initialise ─────────────────────────────────────────────────────
  const selected = [...selectedChildren];
  const selectedIds = new Set(selected.map(s => s.id));

  const candidates = clusterStories
    .filter(s => !selectedIds.has(s.id) && !hiddenIds.has(s.id))
    .sort((a, b) =>
      computeCandidateScore(b, selected) - computeCandidateScore(a, selected)
    );

  // ── Pass 1: evolution role coverage ───────────────────────────────
  const ROLE_PRIORITY: EvolutionRole[] = [
    "first_report", "corroboration", "fact_update", "cause_claim",
    "cause_confirmed", "official_response", "market_reaction",
    "public_reaction", "investigation", "legal_or_regulatory",
    "accountability", "background_context",
  ];

  for (const role of ROLE_PRIORITY) {
    if (selected.length >= cfg.MAX_CHILDREN_PER_PARENT) break;
    const hasRole = selected.some(s => s.evolutionRole === role);
    if (hasRole) continue;

    const best = candidates
      .filter(s => s.evolutionRole === role && !shouldReject(s, selected))
      .sort((a, b) => computeCandidateScore(b, selected) - computeCandidateScore(a, selected))[0];

    if (best) {
      selected.push(best);
      selectedIds.add(best.id);
      hiddenIds.delete(best.id);
    }
  }

  // ── Pass 2: domain diversity ───────────────────────────────────────
  for (const candidate of candidates) {
    if (selected.length >= cfg.MAX_CHILDREN_PER_PARENT) break;
    if (selectedIds.has(candidate.id)) continue;
    if (shouldReject(candidate, selected)) continue;

    const coveredDomains = new Set(selected.map(s => s.sourceContentDomain));
    const coveredTiers   = new Set(selected.map(s => s.temporalTier));

    const addsNewDomain = !coveredDomains.has(candidate.sourceContentDomain);
    const addsNewTier   = candidate.temporalTier && !coveredTiers.has(candidate.temporalTier);

    if (addsNewDomain || addsNewTier) {
      selected.push(candidate);
      selectedIds.add(candidate.id);
      hiddenIds.delete(candidate.id);
    }
  }

  // ── Diagnostics ───────────────────────────────────────────────────
  // Preserve existing diagnostics structure; add new fields
  const debug = (parent.debug as any);
  debug.sourceDiverseSelectionDiagnostics = {
    formulaVersion: "c-e-two-pass-v1-delta-first",
    selectedIds: selected.map(s => s.id),
    coveredTemporalTiers: [...new Set(selected.map(s => s.temporalTier))],
    coveredEvolutionRoles: [...new Set(selected.map(s => s.evolutionRole))],
    coveredDomains: [...new Set(selected.map(s => s.sourceContentDomain))],
    beforeChildCount: selectedChildren.length,
    afterChildCount: selected.length,
  };

  // Tag each selected child with selection reason
  for (const story of selected) {
    (story as any).selectionReason = {
      addedTemporalTier: !selectedChildren.some(s => s.temporalTier === story.temporalTier),
      addedEvolutionRole: !selectedChildren.some(s => s.evolutionRole === story.evolutionRole),
      addedSourceDomain: !selectedChildren.some(s => s.sourceContentDomain === story.sourceContentDomain),
      addedInformationDelta: (story.informationDeltaScore ?? 0) > 0.20,
      scoreBreakdown: {
        deltaScore: story.informationDeltaScore ?? 0.5,
        domainBonus: computeDomainDiversityBonus(story),
        penalty: story.repeatedFactPenalty ?? 0,
      },
    };
  }

  return selected;
}
```

Replace the existing `orderSourceDiverseChildrenForDisplay` (which sorts by angle

```typescript
export function orderChildrenForDisplay(stories: InsightStory[]): InsightStory[] {
  return [...stories].sort((a, b) => {
    // Primary: evolution role order (tells the story)
    const aRoleIdx = EVOLUTION_DISPLAY_ORDER.indexOf(a.evolutionRole ?? "no_new_information");
    const bRoleIdx = EVOLUTION_DISPLAY_ORDER.indexOf(b.evolutionRole ?? "no_new_information");
    if (aRoleIdx !== bRoleIdx) return aRoleIdx - bRoleIdx;

    // Secondary: temporal tier (earlier first within same role)
    const TIER_ORDER: TemporalTier[] = ["breaking","developing","established","reaction","analysis","aftermath"];
    const aTierIdx = TIER_ORDER.indexOf(a.temporalTier ?? "breaking");
    const bTierIdx = TIER_ORDER.indexOf(b.temporalTier ?? "breaking");
    if (aTierIdx !== bTierIdx) return aTierIdx - bTierIdx;

    // Tertiary: information delta (higher = more valuable)
    const aDelta = a.informationDeltaScore ?? 0.5;
    const bDelta = b.informationDeltaScore ?? 0.5;
    if (Math.abs(bDelta - aDelta) > 0.05) return bDelta - aDelta;

    // Quaternary: source authority
    return b.sourceAuthority - a.sourceAuthority;
  });
}
```

**Why this matters for UX:** Currently children show as a flat list of sources.
With arc ordering, the UI tells the train crash story as:
```
T+0    First report     "Train derails near Beed, 2 dead"           [Reuters]
T+2h   Fact update      "Death toll rises to 12, rescue underway"   [NDTV]
T+6h   Official confirm "Railway ministry orders CBI probe"         [Hindu]
T+12h  Market reaction  "IRCTC shares fall 2%"                      [ET]
T+18h  Local impact     "Beed residents demand safety overhaul"     [Lokmat]
```

## UPDATED PIPELINE ORDERING (fixes Agent 1 — CRITICAL)

**File:** `src/insight/src/pipeline/pipeline.ts`

The correct execution order inside `selectTopParentsWithWeakTreeCheck`:

```typescript
for (const parent of ranked) {
  const clusterStories = parent.clusterStoryIds
    .map(id => storiesById.get(id))
    .filter(Boolean) as InsightStory[];

  // ── STEP 1: Assign temporal context (MUST BE FIRST) ──────────────
  const anchor = computeEventAnchor(parent, cfg.MAX_STORY_AGE_HOURS);
  const sortedByTime = [...clusterStories].sort((a, b) => a.publishedAt - b.publishedAt);

  for (const story of clusterStories) {
    story.temporalTier = computeTemporalTier(story.publishedAt, anchor);
  }

  // ── STEP 2: Assign evolution roles ───────────────────────────────
  for (const story of sortedByTime) {
    const priorStories = sortedByTime.filter(s => s.publishedAt < story.publishedAt);
    story.evolutionRole = inferEvolutionRole(story, story.temporalTier!, priorStories);
  }

  // ── STEP 3: Information delta (against chronological baseline) ───
  for (const story of sortedByTime) {
    const baseline = sortedByTime
      .filter(s => s.publishedAt < story.publishedAt)
      .slice(-cfg.MAX_BASELINE_STORIES_FOR_DELTA);
    const delta = extractInformationDelta(story, baseline);
    story.informationDelta = delta;
    story.informationDeltaScore = delta.deltaScore;
    story.repeatedFactPenalty = Math.min(0.25, 1 - delta.deltaScore);
  }

  // ── STEP 4: Post-cluster delta-aware dedup ───────────────────────
  const dedupedCluster = applyPostClusterDeltaDedup(clusterStories, hiddenIds, cfg);

  // ── STEP 5: Build child tree (now uses bridged computeInformationGain) ──
  const children = buildChildTree(parent, dedupedCluster, cfg, hiddenIds);

  // ── STEP 6: C+E two-pass selector ────────────────────────────────
  const sourceDiversityChildren = enforceSourceDiverseChildSelection(
    parent, children, dedupedCluster, cfg, hiddenIds
  );

  parent.childStoryIds = sourceDiversityChildren.map(c => c.id);

  for (const child of sourceDiversityChildren) {
    storiesById.set(child.id, child);
  }

  const clusterIdSet = new Set(parent.clusterStoryIds);
  parent.hiddenDuplicateIds = [...hiddenIds].filter(id => clusterIdSet.has(id));
  parent.weakTree = isWeakTree(sourceDiversityChildren, cfg);
  computePostTreeQualityScore(parent, storiesById, cfg);

  evaluated.push(parent);
}
```
